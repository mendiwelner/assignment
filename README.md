# Sensor Telemetry Explorer

A full-stack app for browsing large sensor telemetry datasets: FastAPI backend, React frontend, SQLite storage, Dockerized.

## Run with Docker (recommended)

```powershell
docker compose up --build
```

- UI: http://localhost:5173
- API docs: http://localhost:8000/docs

After backend code changes:

```powershell
docker compose up --build -d backend
```

## Run locally

1. Create and activate a virtual environment:
   ```powershell
   py -3.12 -m venv .venv
   .\.venv\Scripts\Activate.ps1
   python -m pip install -r backend/requirements.txt
   ```
2. Start the API (from project root):
   ```powershell
   $env:PYTHONPATH = "backend"
   python -m uvicorn backend.app.main:create_app --factory --host 0.0.0.0 --port 8000
   ```
   Or use `.\run_backend.ps1`.
3. In a second terminal, start the frontend:
   ```powershell
   cd frontend
   npm install
   npm run dev
   ```

## Key decisions

- **FastAPI** — async-ready, automatic OpenAPI docs, simple query-param filtering.
- **SQLite** — zero external dependencies, fine for ~1M rows on a single machine; indexes keep filtered reads fast.
- **CSV loader** (`backend/app/loader.py`) — seeds the DB on startup; skips dirty rows (missing fields, bad types).
- **Layered backend** — thin routes, business logic in `services/`, shared filters in `schemas.py`.
- **Indexes** on `asset_id`, `asset_type`, `metric`, `status`, `recorded_at` — used by filtered `/readings` queries.
- **Frontend** — server-side pagination only (DOM never holds the full dataset); 300ms debounce; readings and aggregation load independently so the table stays responsive.

**Assumption:** `recorded_at` is stored/compared as ISO strings. The UI `datetime-local` values are passed through as-is.

## Edge cases considered

- **Invalid time range** (`start > end`) — returns empty results with `total: 0`, no 500 error.
- **Empty filter results** — UI shows an empty state; API returns `items: []`.
- **Huge page numbers** — returns an empty page; `pagination.pages` reflects the real total.
- **Dirty CSV rows** — loader skips rows with missing/invalid fields and logs a count.
- **Aggregation at scale** — live `GROUP BY` over the full table is slow (~30s on unfiltered data); pagination limits response size but not scan cost. See below.

## What I'd do with more time

- **Precomputed aggregation table** — materialize avg/min/max per asset+metric during CSV load (offline/batch), so `/aggregation` queries a small table instead of scanning all readings.
- **Keyset pagination** — replace `OFFSET` with cursor-based paging on `(recorded_at, id)` for deep pages.
- **Docker dev mode** — mount source + `uvicorn --reload` to avoid rebuilding after every change.
- **More tests** — `status=fault` filter, out-of-range page numbers, aggregation with combined filters.
- **Stretch:** pandas ETL (dedupe, UTC normalization), time-series chart with server-side downsampling.

## Data
